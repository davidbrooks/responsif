RWDJS

A Solid Base For Pairing JavaScript and Responsive Web Design

Responsive Mode: Mobile Tablet Desktop

The Setup

RWDJS is super-simple. Just include the rwdjs.css and rwdjs.js files and you're good to go. RWDJS does require jQuery, for the time being. So make sure you're referencing jQuery before rwdjs.js, please.

Once RWDJS takes action it adds a class to the body element, letting you know if it's .responsive_mobile, .responsive_desktop, or .responsive_tablet. It does this by dropping a small div onto the page and watching for changes. So if you resize your page, the body class changes.

What This Script Isn't

Though you could, don't use this script to replace Media Queries. Just don't. (I've done that here, but only to show that the page plugin is working.) If you were to do that, and someone were to shut off JavaScript, or just not have it, or there was a JavaScript error, your page is going to get awkward. So use Media Queries, cool?

What This Script Is

JavaScript doesn't have a sturdy and lightweight way to test what is actually loading the page. (A mobile device, a tablet, or a computer.) So RWDJS creates one by watching how CSS reports changes in Media Queries.

Quick Use-Cases

You can use it to do something like this to load extra content in a desktop view:

$('document').ready(function () {
    var state_test = responsive.run_test();
    if (state_test === 'desktop') {
    // Load that desktop stuff! 
    }
});

Or, if you want to change where in the HTML the navigation sits:

$('document').ready(function(){
    window.onresize = function () {
        if ($('body').is('.responsive_desktop, .responsive_tablet')) {
            // Move your navigation for desktop and tablet users
        } else {
            // Move your navigation for mobile users
        }
    };
});

The possibilities are endless. Enjoy!
